# Handovers: To Backend Developer

## 날짜
2025-12-07

## 현재 상황 (Context)
- 자산 등록/조회 API와 프론트엔드 UI가 완성되었습니다.
- 이제 자산의 평가액을 계산하기 위해 **실시간(또는 최신) 시세 정보**를 가져오는 기능이 필요합니다.

## 해야 할 일 (Tasks)
1. **Price Service 구현 (`app/services/price.py`)**:
   - 외부 API(예: CoinGecko, Yahoo Finance)를 호출하여 자산의 현재가를 가져오는 로직.
   - 개발 단계에서는 무료/공개 API(예: `pycoingecko` 또는 Mock)를 사용.
2. **시세 업데이트 API (`POST /api/v1/prices/refresh`)**:
   - DB에 등록된 모든 자산의 최신 시세를 조회.
   - `Price` 테이블에 새로운 가격 정보를 저장(INSERT).
   - 업데이트된 결과를 반환.
3. **자산 조회 API 개선 (선택)**:
   - `GET /assets` 호출 시, 가장 최신 `Price` 정보도 함께 반환되도록(SQLModel Relationship 활용) 수정 고려.

## 기대 산출물 (Expected Outputs)
- `/prices/refresh` 호출 시 `price` 테이블에 데이터가 쌓임.
- 자산 조회 시 현재 가격을 알 수 있는 구조 마련.
