# Test Plan: Alpha-Sam Project
**Date:** 2025-12-08
**Tester:** QA Lead

## 1. Introduction
This test plan covers the validation of the Alpha-Sam asset management dashboard.
**Scope:**
- Asset Management (CRUD)
- Price Service Integration
- Position Management (Planning phase)
- UI Integration

## 2. Test Scenarios

### Scenario 1: Asset Registration and Position Addition
1. **Asset Registration**
   - Action: `POST /api/v1/assets` with `{"symbol": "BTC", "name": "Bitcoin"}`.
   - Expected: 200 OK, Asset created.
2. **Position Addition** (Pending Implementation)
   - Action: `POST /api/v1/positions` with `{"asset_id": <id>, "quantity": 1.5, "buy_price": 95000}`.
   - Expected: 200 OK, Position created.
3. **Price Refresh**
   - Action: `POST /api/v1/prices/refresh`.
   - Expected: 200 OK, Prices updated.
4. **Verification**
   - Check if profit/loss is calculated correctly (Future).

### Scenario 2: Multi-Asset Management
1. Register multiple assets (ETH, AAPL).
2. Add positions for each.
3. Verify Portfolio Summary calculation.

### Scenario 3: Position Modification (Future)
1. Modify quantity.
2. Delete position.

## 3. Edge Cases
- **Quantity = 0**: Should reject or handle gracefully.
- **Buy Price <= 0**: Should validate/reject.
- **Duplicate Symbol**: Should return 400 Error.
- **Invalid Asset ID**: Position creation should fail.
