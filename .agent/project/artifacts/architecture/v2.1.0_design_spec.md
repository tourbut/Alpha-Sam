# Architecture Specification: v2.1.0 (User Settings & Dashboard Analytics)

## 1. 개요 (Overview)
Alpha-Sam 프로젝트 v2.1.0 업데이트는 사용자의 개인 설정(User Settings) 관리 기능과 포트폴리오의 실적 및 자산 비중을 시각화하는 대시보드 애널리틱스(Dashboard Analytics) 기능을 도입합니다.
기존 v2.0.0의 UUID 마이그레이션 및 다중 포트폴리오 구조를 기반으로 데이터 무결성을 훼손하지 않고 기능 확장에 집중합니다.

## 2. 데이터베이스 스키마 설계 (Database Model Design)

### 2.1. User Settings (사용자 설정)
현재 `app.src.models.user.User` 모델에는 이미 필수 필드(`nickname`, `hashed_password`)가 존재하므로, 스키마의 중대한 수정은 필요하지 않습니다.

**기존 User 모델 (활용 필드)**
- `id`: UUID (Primary Key)
- `email`: 로그인 이메일 (변경 불가)
- `nickname`: 사용자 닉네임 (변경 대상)
- `hashed_password`: 비밀번호 해시 (변경 대상)
- `is_public_leaderboard`: 리더보드 공개 여부 (변경 대상)

### 2.2. Dashboard Analytics (대시보드 분석)
초기 데이터 제공 속도 및 개발 복잡도를 고려하여 파생 테이블(예: 일별 스냅샷) 추가 없이 **런타임 누적 계산(On-the-fly Calculation)** 방식을 채택합니다.
- 자산 비중(Allocation): `app.src.models.position.Position` 뷰 기반 계산
- 가치 추이(History): `app.src.models.transaction.Transaction` 시계열 데이터 누적 합산 로직 사용

## 3. 웹 API 설계 (API Endpoint Specs)

### 3.1. User Settings API
**1. 사용자 프로필 수정**
- **Endpoint**: `PUT /api/v1/users/me/profile`
- **Request (UserUpdate)**:
  ```json
  {
    "nickname": "new_nickname",
    "is_public_leaderboard": true
  }
  ```
- **Response**: 수정된 User 정보 반환 (200 OK)

**2. 사용자 비밀번호 변경**
- **Endpoint**: `PUT /api/v1/users/me/password`
- **Request (PasswordUpdate)**:
  ```json
  {
    "current_password": "old_password123",
    "new_password": "new_password456"
  }
  ```
- **Response**: 성공 여부 메시지 (200 OK)

### 3.2. Dashboard Analytics API
**1. 자산 비중 조회**
- **Endpoint**: `GET /api/v1/analytics/portfolio/{portfolio_id}/allocation`
- **Response List (AssetAllocationResponse)**:
  ```json
  [
    {"ticker": "AAPL", "percentage": 45.5, "total_value": 4550.0},
    {"ticker": "CASH", "percentage": 54.5, "total_value": 5450.0}
  ]
  ```

**2. 포트폴리오 가치 추이 조회**
- **Endpoint**: `GET /api/v1/analytics/portfolio/{portfolio_id}/history`
- **Query Params**: `range` (1W, 1M, 1Y, YTD, ALL)
- **Response List (PortfolioHistoryResponse)**:
  ```json
  [
    {"date": "2026-02-20", "total_value": 10000.0, "uninvested_cash": 2000.0},
    {"date": "2026-02-21", "total_value": 10200.0, "uninvested_cash": 1500.0}
  ]
  ```

## 4. 프론트엔드 UI/UX 설계 (Frontend Design)

### 4.1. Settings 화면 (`/settings` 라우트)
- **Profile 탭**: 닉네임, 리더보드 공개 여부 토글 UI. 저장 시 toast 알림 표시.
- **Security 탭**: 현재 비밀번호와 새로운 비밀번호 입력을 통한 변경 폼 UI. 보안상 확인(Confirm) 필드 제공.

### 4.2. Dashboard 화면 (`/` 라우트 내 통합)
- **Overview Card**: 기존 자산 총액 및 현금 보유량 표시 유지
- **Allocation Chart**: `AssetAllocationChart` 컴포넌트 추가 (Pie/Doughnut 차트). 자산군(주식, 코인, 현금 등) 혹은 종목별 비율 시각화
- **Performance Chart**: `PortfolioPerformanceChart` 컴포넌트 추가 (Line 차트). 기간(range) 필터 버튼 그룹 제공 및 선택에 따른 데이터 리패칭

## 5. 단계별 마일스톤 (Milestone Roadmap)
1. **백엔드 (API & DTO)**: User Settings 수정 API 개발 -> Analytics 통계 쿼리 비즈니스 로직 및 API 엔드포인트 구현 -> 단위 테스트 통과
2. **프론트엔드 (UI & State)**: Settings 페이지 마크업 및 연동 -> Chart 라이브러리(ex: Chart.js) 도입 및 Analytics 컴포넌트 구현
3. **QA 및 배포 준비**: 데이터 누적 오류 점검 (테스트 트랜잭션 수동 인가), UI 반응형 테스트 후 프로덕션 `main` 병합
