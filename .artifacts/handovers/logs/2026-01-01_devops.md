# Handovers: To DevOps

## 날짜
- 2025-12-31

## 현재 상황 (Context)
- v0.9.0 데이터 마이그레이션 및 성능 최적화 작업이 QA 검증(`TC-MIG-01`, `TC-PERF-01`)을 통과했습니다.
- 이제 Production 배포 및 실제 데이터 이관을 진행해야 합니다.

## 해야 할 일 (Tasks)
1. **Source Control**:
   - `develop` 브랜치의 최신 변경사항(마이그레이션 스크립트 포함)을 `main`으로 Merge.
   - Tag `v0.9.0` 생성 및 Push.
2. **Pre-deployment**:
   - **Critical**: Production Database (PostgreSQL) 전체 백업 수행.
   - 백업 파일이 안전하게 저장되었는지 확인.
3. **Deployment**:
   - Backend & Frontend 컨테이너를 `v0.9.0` 버전으로 배포/재기동.
4. **Data Migration**:
   - Production 환경에서 마이그레이션 스크립트 실행:
     `python backend/scripts/migrate_v090_legacy_data.py`
   - 실행 로그에서 "Successfully updated" 메시지 확인.
5. **Monitoring**:
   - 배포 직후 앱 로그(`docker logs`) 모니터링하여 Startup Error 확인.

## 기대 산출물 (Expected Outputs)
- GitHub Repository에 `v0.9.0` 태그 존재.
- Production DB에 마이그레이션 적용 완료 (`owner_id` NULL 데이터 없음).
- 시스템 정상 가동.
