# Handovers: To QA Tester

## 날짜
2025-12-07

## 현재 상황 (Context)
- Alpha-Sam 프로젝트의 기본 기능(Asset 등록, 시세 조회)은 구현되어 있습니다.
- 하지만 Position 기능이 아직 구현되지 않아, 전체 사용자 시나리오를 테스트할 수 없는 상태입니다.
- 백엔드와 프론트엔드 개발이 진행 중이므로, 각 단계별로 테스트 계획을 수립하고 준비해야 합니다.

## 해야 할 일 (Tasks)
1. **테스트 환경 설정**
   - Docker Compose를 사용하여 로컬 테스트 환경 구축.
   - 테스트용 데이터베이스 초기화 방법 확인.
   - 테스트 시나리오를 위한 샘플 데이터 준비 (Asset, Position, Price).

2. **핵심 사용자 시나리오 테스트 계획 수립**
   - 다음 시나리오에 대한 테스트 케이스 작성:
     - **시나리오 1: 자산 등록 및 포지션 추가**
       1. Asset 등록 (Symbol: BTC, Name: Bitcoin)
       2. Position 추가 (Quantity: 1.5, Buy Price: 95000)
       3. 시세 새로고침
       4. 수익률/평가액이 올바르게 계산되어 표시되는지 확인
     - **시나리오 2: 여러 자산 관리**
       1. 여러 Asset 등록 (BTC, ETH, AAPL)
       2. 각각에 Position 추가
       3. 포트폴리오 요약이 올바르게 계산되는지 확인
     - **시나리오 3: 포지션 수정/삭제**
       1. 기존 Position 수정 (수량 변경)
       2. 수익률이 재계산되는지 확인
       3. Position 삭제 후 Asset만 남는지 확인

3. **에지 케이스 테스트 계획**
   - `quantity = 0`인 경우 처리 확인
   - `buy_price <= 0`인 경우 유효성 검증 확인
   - 시세가 없는 Asset의 수익률 계산 시도 시 에러 처리 확인
   - 중복 Symbol 등록 시 에러 처리 확인
   - 존재하지 않는 Asset에 Position 추가 시도 시 에러 처리 확인

4. **API 엔드포인트 테스트**
   - FastAPI 자동 생성 문서 (`/docs`)를 사용하여 각 엔드포인트 테스트.
   - 다음 엔드포인트 검증:
     - `GET /api/v1/assets` - Asset 목록 조회
     - `POST /api/v1/assets` - Asset 생성
     - `GET /api/v1/positions` - Position 목록 조회
     - `POST /api/v1/positions` - Position 생성
     - `POST /api/v1/prices/refresh` - 시세 새로고침
   - 요청/응답 형식이 스키마와 일치하는지 확인.

5. **프론트엔드 UI 테스트**
   - 브라우저에서 실제 사용자 행동 시뮬레이션.
   - Asset 추가 모달 동작 확인.
   - Position 추가 모달 동작 확인 (구현 후).
   - 에러 메시지가 사용자 친화적으로 표시되는지 확인.
   - 로딩 상태가 적절히 표시되는지 확인.

6. **통합 테스트 리포트 작성**
   - 테스트 결과를 `.artifacts/qa_reports/` 디렉토리에 기록.
   - 발견된 버그는 재현 절차, 기대 결과, 실제 결과를 명확히 기록.
   - 스크린샷 첨부 (필요 시).

7. **성능 테스트 (선택적)**
   - 많은 수의 Asset/Position이 있을 때 페이지 로딩 속도 확인.
   - 시세 새로고침 시 응답 시간 확인.

## 기대 산출물 (Expected Outputs)
- 테스트 계획 문서 (`.artifacts/qa_reports/test_plan_YYYY-MM-DD.md`)
- 핵심 시나리오 테스트 결과 리포트

- 발견된 버그 리포트 (있는 경우)
