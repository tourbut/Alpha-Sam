# Implementation Plan - v0.8.0 (Authentication System)

## 개요
v0.8.0은 `FastAPI-Users`와 `JWT`를 도입하여 안전하고 표준화된 인증 시스템을 구축하는 것을 목표로 합니다.
기존 `X-User-Id` 헤더 기반의 시뮬레이션 방식을 점진적으로 제거하기 위해, **Phase 1(Hybrid)** 단계에서는 두 방식을 모두 지원하고, **Phase 2(Cleanup)** 에서 시뮬레이션 로직을 제거합니다.

## Phase 1: Authentication Infrastructure & Hybrid Mode
**목표:** `FastAPI-Users` 설정, User 모델 마이그레이션, 그리고 구/신 인증 방식이 공존하는 하이브리드 모드 구현.

### 1.1 Dependency & Configuration
- **Backend**:
  - `pyproject.toml`: `fastapi-users[sqlalchemy]`, `passlib[bcrypt]` 등 의존성 추가.
  - `src/core/config.py`: `SECRET_KEY`, `ALGORITHM`, `ACCESS_TOKEN_EXPIRE_MINUTES` 등 설정 추가.

### 1.2 User Model Migration
- **파일**: `src/models/user.py`
- **변경**:
  - `FastAPI-Users`의 `SQLModelBaseUserDB` 상속.
  - `email`(unique), `hashed_password`, `is_active`, `is_superuser`, `is_verified` 필드 표준화.
  - `schemas.py`: `UserRead`, `UserCreate`, `UserUpdate` 스키마 정의.

### 1.3 Auth Component Setup (`src/core/security.py` or `src/users.py`)
- **UserManager**: 사용자 생성 시 `NotificationSettings` 자동 생성 로직 hook(`on_after_register`) 구현.
- **Strategy**: `JWTStrategy` 설정.
- **Backend**: `BearerTransport` + `JWTStrategy` 조합의 `AuthenticationBackend` 구성.

### 1.4 API Endpoints (`src/api/v1/auth.py`)
- `FastAPI-Users`가 제공하는 Router 포함.
  - `/auth/register`
  - `/auth/jwt/login`
  - `/auth/jwt/logout`
  - `/users/me` (기존 endpoints와 경로 충돌 주의, `/users/me`로 통일 권장)

### 1.5 Hybrid Dependency (`src/api/deps.py`)
- **기능**: `get_current_user` 의존성 수정.
- **로직**:
  1. `Authorization` 헤더(Bearer JWT) 확인 -> 유효하면 User 객체 반환.
  2. 없다면 `X-User-Id` 헤더 확인 -> 유효하면(DB 조회) User 객체 반환 및 **Warning Log** 기록.
  3. 둘 다 없으면 `401 Unauthorized`.

## Phase 2: Frontend Integration
**목표:** 프론트엔드에서 로그인/회원가입 UI를 제공하고, 토큰 기반으로 API를 호출합니다.

### 2.1 UI Pages
- **Pages**:
  - `/login`: 이메일/비밀번호 입력 폼.
  - `/register`: 회원가입 폼.
  - `/settings`: 기존 설정 페이지에 '로그아웃' 버튼 추가.

### 2.2 Auth Store & Client
- **Store**: Svelte Store를 사용하여 `accessToken`, `user` 정보 관리.
- **API Client**: `axios` 또는 `fetch` wrapper 수정.
  - Request Interceptor: Store에 토큰이 있으면 `Authorization: Bearer <token>` 헤더 추가.
  - `X-User-Id` 헤더 전송 로직은 당분간 유지(Hybrid 테스트용)하되, 토큰이 우선순위를 갖도록 설정.

---

## Phase 3: Cleanup (v0.8.0-final)
**목표:** 시뮬레이션 헤더 로직 제거. (이 단계는 Phase 1, 2 안정화 후 진행)

- `deps.py`: `X-User-Id` 체크 로직 삭제.
- Frontend: `X-User-Id` 전송 로직 삭제.

---

## 검증 계획 (Verification Plan)

### Automated Tests
1. **Model Test**: `User` 모델 생성 및 필드 제약조건(Email Unique 등) 테스트.
2. **Auth Integration**:
   - `/register` 엔드포인트로 유저 생성.
   - `/login` 후 JWT 토큰 발급 확인.
   - 발급된 토큰으로 `/users/me` 접근 성공 확인.
3. **Hybrid Logic Test**:
   - JWT 없이 `X-User-Id` 만으로 `/users/me` 접근 성공 확인 (Phase 1).
   - 둘 다 없이 접근 시 401 확인.

### Manual Verification
1. **Swagger UI Flow**:
   - `Authorize` 버튼을 통해 JWT 입력 후 API 호출 테스트.
2. **Frontend Flow**:
   - 로그인 페이지 -> 로그인 성공 -> 메인 대시보드 리다이렉트.
   - 로컬 스토리지 확인 (토큰 저장 여부).
   - 로그아웃 -> 로그인 페이지 리다이렉트 & 저장소 클리어.
